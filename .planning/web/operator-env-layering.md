# Operator-env layering — `web-deploy/.env` (Phase 1.5)

Date: 2026-05-07
Branch: `VarLens-Web`
Status: shipped

Tightens the operator interface so all "secrets the operator types"
land in one gitignored file at `web-deploy/.env`, rather than three
shell exports and one tracked-file edit.

## Two layers, made physical

- **Layer 1 — runtime config Docker Compose consumes** at
  `web-deploy/compose/.env` (server-only, generated by `stack-up`).
  Docker auto-loads `.env` from the compose directory; moving it would
  break every IDE plugin and convention. Stays put.
- **Layer 2 — operator-typed secrets** at `web-deploy/.env`
  (gitignored already by `.env` rule; 0600). Single source for
  `GHCR_TOKEN`, `RESTIC_S3_ACCESS_KEY`, `RESTIC_S3_SECRET_KEY`,
  `VARLENS_ADMIN_USERNAME`, `VARLENS_ADMIN_PASSWORD`,
  `VARLENS_ADMIN_DISPLAY_NAME`.
- **Tofu** (`hcloud_token`, `ssh_pubkey`) stays in
  `tofu/environments/pilot/terraform.tfvars` — different lifecycle
  (provider config, not bring-up secret).

## What changed

- New `web-deploy/.env.example` — Layer 2 template.
- `pilot.sh` sources `web-deploy/.env` early, before preflight.
  Shell exports still override (CI / one-off use unchanged).
- `pilot.sh` preflight added a `VARLENS_ADMIN_*` warn-when-missing
  check; previously the app booted with no admin and the operator
  discovered it at first login.
- `pilot.sh` success banner walks the operator through capturing the
  recovery key and blanking the password value post-bootstrap.
- `Makefile stack-up` injects `VARLENS_ADMIN_*` into the server-side
  `/mnt/data/app/.env` on **first boot only**, via SSH stdin (avoids
  `ps` exposure and sed-escape issues with arbitrary password chars).
  Re-runs are no-ops; `server.ts` already ignores post-bootstrap env
  rotation with a WARN.
- `compose/.env.example` keeps the keys (deploy-stack gate substring
  check) but the comment now points to `web-deploy/.env` and forbids
  filling them in here.
- `DEPLOY.md` flow rewritten: edit one file, run one command.

## Out of scope (next deltas)

- `varlens admin rotate` — proper rotation path, not env-based.
- Encrypting `web-deploy/.env` at rest (SOPS+age is already wired for
  `secrets/*.yaml`; same recipients would extend cleanly).
