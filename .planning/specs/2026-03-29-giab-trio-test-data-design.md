**Date:** 2026-03-29

## 1. Goal

Produce a small, committed set of VCF test files derived from the GIAB Chinese Trio (HG005/HG006/HG007) that:

1. Cover real-world VCF parsing (multi-sample, genotype fields, QUAL/FILTER)
2. Include VEP CSQ annotations (Docker + cache)
3. Include SnpEff ANN + SnpSift ClinVar annotations (conda)
4. Contain cherry-picked variants for inheritance filter edge cases
5. Stay small enough (< 1 MB total) to commit directly to the repo
6. Are reproducible via a documented, idempotent script

## 4. Region Selection & Edge Case Strategy

### 4.1 Primary Region

**`chr22:20,000,000-21,000,000`** (1 MB window)

- Within GIAB high-confidence regions for all three samples
- Contains protein-coding genes in the 22q11 region
- Expected yield: ~1,000-2,000 variants per sample

### 4.2 Cherry-Picked Edge Cases

Extracted from the full chr22 merged trio VCF using `bcftools query` genotype expressions, then combined into `edge-cases.vcf.gz`.

| Category | Selection criteria | Purpose | Target count |
|----------|-------------------|---------|--------------|
| Multi-allelic | Sites with 2+ ALT alleles | Allele splitting logic | 3-5 |
| Indels | Small insertions + deletions (1-50 bp) | Indel parsing, left-alignment | 5-8 |
| Missing genotype | `./.` in at least one sample | No-call handling | 3-5 |
| De novo candidate | Son 0/1, Father 0/0, Mother 0/0 | De novo inheritance filter | 3-5 |
| Hom recessive candidate | Son 1/1, Father 0/1, Mother 0/1 | AR-Hom filter | 3-5 |
| Compound het candidate | Son 0/1 at 2+ sites in same gene, each from different parent | CompHet filter | 2 pairs (4 variants) |
| Autosomal dominant | Son 0/1, one parent 0/1, other 0/0 | AD filter | 3-5 |
| High-impact | Stop-gain, frameshift, splice (from annotation) | Consequence filtering | 3-5 |
| FILTER != PASS | Filtered sites (if present in GIAB) | FILTER column handling | 2-3 |

**chrX / X-linked:** GIAB benchmark VCFs may not include chrX. If unavailable, X-linked test variants are included in `synthetic-unit-test.vcf` with hand-crafted genotypes.

**Annotation of edge cases:** `edge-cases.vcf.gz` is kept **unannotated** (raw genotypes only). Its purpose is testing genotype-pattern-based inheritance filters, not annotation parsing. Annotated edge cases for parser testing are covered by `synthetic-unit-test.vcf` which contains hand-crafted CSQ and ANN fields.

### 4.3 Synthetic Unit Test VCF

`synthetic-unit-test.vcf` is a hand-crafted, uncompressed VCF (~20 variants) generated by the script. It covers parser edge cases that may not appear in real GIAB data:

- VEP CSQ format header + variants with CSQ annotations
- SnpEff ANN format header + variants with ANN annotations
- Multi-transcript annotations (3+ transcripts per variant)
- Empty/missing annotation subfields
- chrX variants with hemizygous male genotypes (0 and 1)
- Mixed ploidy genotype representations
- Unicode/special characters in gene names (if applicable)
- Both `PASS` and non-`PASS` FILTER values

This file has **no external data dependency** and is always regeneratable.

## 6. Preparation Script Design

### 6.1 Script: `scripts/prepare-test-data.sh`

**Interface:**

```
Usage:
  ./scripts/prepare-test-data.sh [OPTIONS]

Options:
  --step STEP       Run a specific step or comma-separated list:
                      download, subset, merge, cherry-pick,
                      annotate-vep, annotate-snpeff, synthetic,
                      package, verify, all (default)
  --workdir DIR     Working directory for intermediates
                      (default: ~/data/varlens-testdata)
  --outdir DIR      Final output committed to repo
                      (default: tests/test-data/vcf)
  --region REGION   Primary extraction region
                      (default: chr22:20000000-21000000)
  --cleanup         Remove intermediate files after packaging
  --force           Regenerate outputs even if they already exist
  --dry-run         Print steps without executing
  --help            Show usage
```

**Design principles:**
- **Idempotent** — each step checks if output exists, skips if present. Pass `--force` to regenerate.
- **Resumable** — downloads use `wget -c`, intermediate files preserved
- **Logged** — timestamped output with step headers, saved to `$WORKDIR/logs/`
- **Prerequisite validation** — checks conda env, Docker, disk space, VEP cache before any work

### 6.2 Step Pipeline

```
Step 1: download
  └─ Download GIAB benchmark VCFs + .tbi for HG005, HG006, HG007

Step 2: subset
  └─ bcftools view -r chr22 for each sample
  └─ Rename samples if needed (bcftools reheader)

Step 3: merge
  └─ bcftools merge → trio chr22 VCF
  └─ Extract single-sample (HG005 only)

Step 4: cherry-pick
  └─ bcftools query to find edge case variants by genotype pattern
  └─ bcftools view -R to extract matching positions
  └─ Merge into edge-cases.vcf.gz

Step 5: annotate-vep
  └─ Region VCF → docker run ensembl-vep --everything --vcf --offline
  └─ Edge-cases VCF → same VEP command
  └─ bgzip + tabix output

Step 6: annotate-snpeff
  └─ Region VCF → snpEff ann | SnpSift annotate (ClinVar)
  └─ Edge-cases VCF → same pipeline
  └─ bgzip + tabix output

Step 7: synthetic
  └─ Generate synthetic-unit-test.vcf (pure bash heredoc, no tools needed)
  └─ Hand-crafted variants with CSQ + ANN headers and data

Step 8: package
  └─ Copy final files to --outdir
  └─ Generate PED file
  └─ Generate README.md with provenance (tool versions, variant counts, md5sums)

Step 9: verify
  └─ Validate all output files exist and are non-empty
  └─ bcftools stats on each VCF (variant count, sample count)
  └─ Check CSQ/ANN headers present in annotated files
  └─ Print summary table
```

### 6.3 Cherry-Pick Queries

```bash
# De novo candidates: son=0/1, father=0/0, mother=0/0
bcftools query -f '%CHROM\t%POS\n' \
  -i 'GT[0]="0/1" && GT[1]="0/0" && GT[2]="0/0"' \
  trio_chr22.vcf.gz | head -5

# Hom recessive: son=1/1, both parents=0/1
bcftools query -f '%CHROM\t%POS\n' \
  -i 'GT[0]="1/1" && GT[1]="0/1" && GT[2]="0/1"' \
  trio_chr22.vcf.gz | head -5

# Autosomal dominant: son=0/1, one parent=0/1, other=0/0
bcftools query -f '%CHROM\t%POS\n' \
  -i 'GT[0]="0/1" && ((GT[1]="0/1" && GT[2]="0/0") || (GT[1]="0/0" && GT[2]="0/1"))' \
  trio_chr22.vcf.gz | head -5

# Multi-allelic: more than one ALT
bcftools query -f '%CHROM\t%POS\n' \
  -i 'N_ALT>1' \
  trio_chr22.vcf.gz | head -5

# Missing genotypes
bcftools query -f '%CHROM\t%POS\n' \
  -i 'GT[0]="./." || GT[1]="./." || GT[2]="./."' \
  trio_chr22.vcf.gz | head -5

# Indels
bcftools query -f '%CHROM\t%POS\n' \
  -i 'TYPE="indel"' \
  trio_chr22.vcf.gz | head -5
```

Compound het candidates require post-annotation gene grouping — extracted after VEP/SnpEff annotation using gene symbol from CSQ/ANN.

## 9. Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| GIAB FTP down or slow | `wget -c` for resume; downloads cached in workdir |
| VEP cache download fails (20 GB) | Separate setup script; can retry independently |
| Cherry-pick queries find 0 matches for a category | Log warning, skip category; synthetic VCF covers the gap |
| GIAB has no chrX data | X-linked variants in synthetic VCF only |
| SnpEff database version mismatch | Pin `GRCh38.mane.1.2.ensembl`; document exact version |
| Annotation adds too much data (file > 1 MB) | Use `--pick` for VEP (one transcript per variant) on region file; keep multi-transcript only in edge cases |
| bcftools merge creates unexpected records | Verify step checks sample count and variant counts post-merge |
