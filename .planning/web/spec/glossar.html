Glossary

Terms used in vertrag.html, infrastruktur.html, app.html, and fahrplan.html. Cross-plan, maintained in one place.

Term Meaning
ADR Architecture Decision Record. Versioned decision documentation. Immutable; a new ADR replaces an old one (status: superseded by ADR-X).
age File-encryption tool. Key provider for SOPS.
ALB / ACM / WAF / GuardDuty / EBS / EFS / KMS / RDS AWS services (Application Load Balancer, Certificate Manager, Web Application Firewall, threat detection, block storage, file system, Key Management Service, managed PostgreSQL).
Argon2 Password hashing function. Used in Application Phase 1 for the single-user login.
Data Processing Agreement (DPA) Agreement under GDPR Article 28 between controller and processor.
B3S Industry-specific security standard for healthcare.
Caddy Reverse proxy with automatic TLS management. Fulfils Interface 3 in the Hetzner variant.
cloud-init Linux first-boot mechanism that processes a YAML configuration.
cosign / Sigstore / SLSA Signing stack for the container-image supply chain.
Data Protection Impact Assessment (DPIA) Mandatory document under GDPR Article 35 in case of high risk.
Fastify / Pino / Vue 3 / Vuetify 4 / Vite / Pinia / oidc-client-ts Components of the application stack.
GitHub Container Registry (GHCR) Registry for container images with SemVer tags.
Identity Provider (IdP) OIDC-compliant component for authentication.
ISO 15189 Standard for the accreditation of medical laboratories.
KEMP Hardware load balancer; listed as a candidate in the stakeholder's web-access logbook.
Logbook Stakeholder-internal logbook on the target architecture for web access, MFA, and VPN. ADR-5 defers to it.
LUKS2 Linux Unified Key Setup, encryption at the block-device layer.
Medical Device Regulation (MDR) EU Regulation 2017/745. Applies to software used for primary diagnostics.
MIT License that permits adoption by other laboratories.
Netdata / Diun Single-host monitoring; watcher for container-image updates.
OpenID Connect (OIDC) / Proof Key for Code Exchange (PKCE) Authentication protocol on top of OAuth 2.0; secure browser flow for public clients.
Recovery Point/Time Objective (RPO/RTO) Maximum tolerable data loss and maximum recovery time, respectively.
Row-Level Security (RLS) Postgres feature that hides rows depending on the user.
Software Bill of Materials (SBOM) / syft Listing of all dependencies of a build; the corresponding generator.
SOPS Per-value encryption inside YAML and JSON files, combined with age as the key provider.
Sovereign Cloud Cloud region with extended guarantees on data sovereignty (operations exclusively within the EU, no access by non-European authorities without a legal basis).
TOM Technical and organisational measures under GDPR Article 32.
Trivy CVE scanner for container images.
Web-Access Logbook See Logbook.
