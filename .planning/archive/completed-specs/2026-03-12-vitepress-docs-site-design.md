# VitePress Documentation Site Design

**Issue:** [#52](https://github.com/berntpopp/VarLens/issues/52)
**Date:** 2026-03-12
**Status:** Approved

## Overview

Add a VitePress documentation site for VarLens with automated Playwright screenshots, a synthetic demo dataset, and GitHub Pages deployment on release tags.

## Key Decisions

- **Approach:** Monorepo-integrated VitePress in `docs/` (existing `docs/plans/` and `docs/superpowers/` move to `.planning/`)
- **Deployment:** GitHub Pages at `berntpopp.github.io/VarLens/`, triggered on release tags (`v*.*.*`)
- **Theme:** VitePress default with light/dark mode, modern styling
- **Screenshots:** Automated via Playwright against compiled Electron app with synthetic demo data
- **Content scope:** Core pages with real content (not placeholders) — installation, importing data, key features with screenshots
- **Reference implementation:** [gnomad-carrier-frequency docs](https://github.com/berntpopp/gnomad-carrier-frequency/tree/main/apps/web/docs)

## 3. Automated Screenshot Pipeline

Dedicated Playwright test: `tests/e2e/screenshots.e2e.ts`

### Workflow

1. Launch Electron app via `_electron`
2. Import demo case via `electronApp.evaluate()` — call the ImportService directly in the main process context (bypasses file dialog). The existing E2E tests use this same pattern.
3. Navigate to views, wait for data-dependent selectors (e.g., `.v-data-table`, specific row counts) before capturing
4. Save screenshots to `docs/public/screenshots/` (same directory in both local and CI runs)

### Screenshots

| Screenshot | View | Filename |
|------------|------|----------|
| Empty state | Fresh app, no cases | `empty-state.webp` |
| Import dialog | Import in progress | `importing-data.webp` |
| Case list | Sidebar with demo case | `case-list.webp` |
| Variant table | Full table with data | `variant-table.webp` |
| Filter toolbar | Filters applied | `filters-active.webp` |
| Column filters | Per-column text filter | `column-filters.webp` |
| Variant detail panel | Side panel open | `variant-details.webp` |
| ACMG classification | Evidence editor | `acmg-classification.webp` |
| Annotations | Star + comment + tags | `annotations.webp` |
| Cohort view | Cohort table | `cohort-view.webp` |
| Dark mode | Variant table dark | `dark-mode.webp` |

### Technical Details

- **Viewport:** Fixed 1280x800 for consistency
- **Format:** WebP (or PNG if webp conversion adds complexity)
- **Idempotent:** No timestamps or dynamic data in screenshots
- **IPC direct:** Uses `electronApp.evaluate()` for import, dark mode toggle, etc.

## 4. GitHub Actions Deployment

New workflow: `.github/workflows/docs.yml`

**Trigger:** Release tags (`v*.*.*`)

### Jobs

```
1. build-and-screenshot (ubuntu-latest):
   - Checkout, install deps, rebuild for Electron
   - electron-vite build
   - xvfb-run playwright test tests/e2e/screenshots.e2e.ts
   - Upload screenshots as artifact

2. deploy-docs (needs: build-and-screenshot):
   - Download screenshots into docs/public/screenshots/
   - npm run docs:build
   - Deploy to GitHub Pages via actions/deploy-pages
```

**GitHub Pages config:** Repository Settings > Pages > Source: GitHub Actions.

**Required workflow permissions:** `pages: write`, `id-token: write` (for `actions/deploy-pages`). Add `concurrency` group to prevent race conditions on rapid successive releases.

**No changes** to existing `build.yml` or `release.yml` workflows.

## 5. Content Strategy

### Landing Page

Hero with VarLens name/tagline. "Download" + "Getting Started" action buttons. 4-5 feature cards: variant analysis, filtering, ACMG classification, cohort analysis, offline/secure.

### Guide (3 pages)

- **Introduction:** What VarLens is, target audience, offline-first architecture, key capabilities
- **Installation:** Download links per platform, first launch, database creation
- **Importing data:** Step-by-step with screenshots, supported formats, import progress

### Features (5 pages)

- **Variant table:** Column customization, sorting, pagination, row selection
- **Filtering:** Filter toolbar, per-column text filters, FTS search
- **Variant details:** Side panel sections, VEP enrichment, external links
- **Annotations:** Stars, comments, tags, ACMG evidence editor
- **Cohort analysis:** Cohort mode, carrier aggregation, gene burden

Each page: brief explanation, screenshot, step-by-step usage notes. Concise guided tour, not a manual.

### Reference (3 pages)

Supported formats (specs with examples), keyboard shortcuts, FAQ.

### About (4 pages)

Overview, citation, changelog, contributing (dev setup, PR workflow).
